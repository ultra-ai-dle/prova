## 시나리오 1: 첫 방문 자동 시작

- [ ] **자동 시작** — 첫 방문(localStorage 비어있음) + Pyodide 로딩 완료 → 500ms 후 투어가 자동 시작된다
- [ ] **오버레이 렌더링** — 투어 시작 시 → 반투명 검정 오버레이가 전체 화면을 덮고, 헤더 영역만 spotlight 컷아웃으로 보인다
- [ ] **1단계 내용** — 첫 화면에 "Prova에 오신 것을 환영합니다" 제목과 안내 본문이 표시된다
- [ ] **진행 표시** — 툴팁 상단에 녹색 프로그레스 바 + "1 / 7" 스텝 표시가 보인다

## 시나리오 2: 스텝별 순방향 탐색

- [ ] **Next 클릭 (1→2)** — Next 버튼 클릭 → spotlight이 코드 에디터로 이동하고 "코드 에디터" 설명이 표시된다
- [ ] **Next 클릭 (2→3)** — Next 클릭 → spotlight이 언어 선택 드롭다운으로 이동, "언어 선택" 설명 표시
- [ ] **Next 클릭 (3→4)** — Next 클릭 → spotlight이 입력 영역으로 이동, "입력 & 실행" 설명 표시
- [ ] **Next 클릭 (4→5)** — Next 클릭 → spotlight이 시각화 패널로 이동, "시각화 패널" 설명 표시
- [ ] **Next 클릭 (5→6)** — Next 클릭 → spotlight이 디버그 컨트롤로 이동, "디버그 컨트롤" 설명 표시
- [ ] **Next 클릭 (6→7)** — Next 클릭 → spotlight이 변수 모니터로 이동, "변수 모니터" 설명 표시
- [ ] **마지막 스텝 버튼** — 7단계에서 Next 버튼 텍스트가 "시작하기!"로 변경된다
- [ ] **spotlight 애니메이션** — 스텝 전환 시 clip-path가 300ms ease-out으로 부드럽게 이동한다

## 시나리오 3: 역방향 탐색

- [ ] **Prev 클릭** — 3단계에서 Prev 클릭 → 2단계(코드 에디터)로 돌아간다
- [ ] **1단계 Prev 숨김** — 1단계에서는 Prev 버튼이 표시되지 않는다

## 시나리오 4: 키보드 내비게이션

- [ ] **ArrowRight / Enter** — 투어 활성 상태에서 → / Enter 입력 → 다음 스텝으로 이동
- [ ] **ArrowLeft** — 투어 활성 상태에서 ← 입력 → 이전 스텝으로 이동
- [ ] **Escape** — 투어 활성 상태에서 Escape 입력 → 투어 즉시 종료 + localStorage에 완료 기록

## 시나리오 5: 투어 완료

- [ ] **마지막 스텝 완료** — 7단계에서 "시작하기!" 클릭 → 오버레이/툴팁 사라지고, `prova:tourCompleted`가 localStorage에 저장된다
- [ ] **재방문 시 미시작** — 투어 완료 후 새로고침 → 투어가 자동 시작되지 않는다

## 시나리오 6: 스킵

- [ ] **Skip 클릭** — 어느 스텝에서든 "건너뛰기" 클릭 → 투어 즉시 종료 + localStorage에 완료 기록
- [ ] **스킵 후 재방문** — 스킵 후 새로고침 → 투어가 자동 시작되지 않는다

## 시나리오 7: 수동 재시작

- [ ] **헤더 버튼 트리거** — 투어 완료 후 → 헤더 우측 설정 아이콘 클릭 → 투어가 1단계부터 다시 시작된다
- [ ] **실행 중에도 트리거** — 코드 실행이 완료된 상태(시각화 표시 중)에서 → 설정 아이콘 클릭 → 투어가 정상 시작된다

## 시나리오 8: 디버깅 시작 시 자동 종료

- [ ] **투어 중 실행** — 투어 3단계 진행 중 → 코드 실행(Run 클릭) → uiMode가 "running"으로 전환되면서 투어가 자동 종료된다
- [ ] **자동 종료 후 상태** — 투어가 자동 종료된 후 → 오버레이/툴팁이 완전히 사라지고 실행이 정상 진행된다

## 엣지 케이스

- [ ] **브라우저 리사이즈** — 투어 진행 중 브라우저 크기 변경 → spotlight 위치와 툴팁 위치가 새 레이아웃에 맞게 재계산된다
- [ ] **타겟 요소 미발견** — data-tour 속성이 있는 요소가 DOM에 없는 경우 → 투어가 렌더링되지 않는다 (크래시 없음)
- [ ] **빠른 연타** — Next 버튼을 빠르게 연타 → 스텝이 순서대로 정상 이동하고 마지막 스텝에서 멈춘다
- [ ] **localStorage 접근 불가** — 시크릿 모드 등에서 localStorage 접근 실패 → try/catch로 크래시 없이 동작, 매 방문마다 자동 시작

## UI 상태

- [ ] **z-index 우선순위** — 투어 오버레이(z-60)와 툴팁(z-61)이 → 토스트, 모달 등 다른 UI 위에 표시된다
- [ ] **프로그레스 바** — 스텝 진행에 따라 → 녹색 바가 1/7 ~ 7/7로 채워진다 (width transition 300ms)
- [ ] **툴팁 방향** — 각 스텝의 placement에 따라 → 화살표가 올바른 방향(bottom-center, right, left, bottom-left)을 가리킨다
- [ ] **배경 인터랙션 차단** — 오버레이가 활성화된 동안 → 배경 요소(에디터, 버튼 등)를 클릭할 수 없다
