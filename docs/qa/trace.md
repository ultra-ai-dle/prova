# QA: Trace 병합

> 마지막 업데이트: 2026-04-11
> 대상 파일: src/features/trace/merge.ts

## 정상 동작
- [ ] **기본 병합** — 코드 실행 후 AI 분석이 완료되면 → 각 스텝에 설명 텍스트와 시각화 액션이 함께 표시된다
- [ ] **점진적 스트리밍** — AI 설명이 8스텝 단위로 도착할 때마다 → 해당 스텝의 설명이 즉시 반영되어 슬라이더를 움직이면 확인할 수 있다
- [ ] **재실행 시 초기화** — 코드를 수정하고 다시 실행하면 → 이전 스텝 설명이 모두 초기화되고 슬라이더가 0으로 돌아간다

## 엣지 케이스
- [ ] **AI 응답 부족** — AI가 실행 스텝보다 적은 수의 설명을 반환하면 → 나머지 스텝은 설명 없이 빈 상태로 표시되지만 시각화는 정상 동작한다
- [ ] **AI 응답 초과** — AI가 실행 스텝보다 많은 설명을 반환하면 → 초과분은 무시되고 실행 스텝 수만큼만 표시된다

## 에러 케이스
- [ ] **런타임 에러 스텝** — 코드 실행 중 에러가 발생한 스텝에서 → 에러 라인이 빨간색으로 표시되고 출력 패널에 에러 타입과 메시지가 출력된다
- [ ] **AI 에러 정보** — AI가 에러 원인을 분석하면 → 해당 스텝에서 `root_cause`와 `fix_hint`가 함께 표시된다
- [ ] **AI 전체 실패** — AI 설명 생성이 완전히 실패하면 → 모든 스텝에 "L.N 실행" 형태의 기본 설명이 표시되고 시각화는 `highlight` 액션으로 동작한다

## UI 상태
- [ ] **분석 대기** — 실행 완료 후 AI 분석 전 → 시각화 패널에 "코드 분석중... AI 응답을 기다리는 중입니다." 스피너가 표시된다
- [ ] **에러 스텝 자동 이동** — 런타임 에러가 있는 트레이스에서 → 자동으로 에러 스텝으로 이동하고 `errorStep` 모드로 전환된다
- [ ] **자동 재생 중 에러 스텝** — 자동 재생 중 에러 스텝에 도달하면 → 재생이 자동으로 멈추고 에러 스텝에서 정지한다
