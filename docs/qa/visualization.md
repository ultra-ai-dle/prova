# QA: Visualization

> 마지막 업데이트: 2026-04-11
> 대상 파일: src/features/visualization/GridLinearPanel.tsx, src/features/visualization/GraphPanel.tsx, src/features/visualization/ThreeDVolumePanel.tsx, src/features/visualization/linearPointerHelpers.ts

## 정상 동작

### Grid/Linear 패널
- [ ] **2D 그리드 표시** — BFS/DFS 등 2D 배열 알고리즘을 실행하면 → 행(y)/열(x) 레이블이 있는 격자가 표시되고 현재 위치에 초록색 `→` 화살표가 나타난다
- [ ] **현재 셀 하이라이트** — 스텝을 이동하면 → 현재 탐색 중인 셀이 초록색 테두리로 강조된다
- [ ] **변경된 셀 표시** — 이전 스텝과 값이 달라진 셀은 → 파란색 테두리와 배경으로 표시된다
- [ ] **1D 배열 표시** — LINEAR 전략의 알고리즘을 실행하면 → 가로 스크롤 가능한 셀 행이 인덱스 레이블과 함께 표시된다
- [ ] **포인터 배지** — Two-pointer 알고리즘에서 → 배열 셀 아래에 색상 구분된 포인터 배지가 표시된다
- [ ] **빈 배열** — 배열이 비어 있으면 → "배열이 비어 있습니다" 메시지가 표시된다
- [ ] **비트마스크 모드** — BIT ON 토글을 활성화하면 → 정수 값이 이진수 문자열로 표시된다
- [ ] **알고리즘 컨텍스트** — 관련 변수가 있으면 → 배열 위에 `key=value` 형태의 컨텍스트 라인이 표시된다

### 그래프 패널
- [ ] **그래프 시각화** — 그래프 알고리즘을 실행하면 → 노드와 엣지가 D3 force 시뮬레이션으로 배치된다
- [ ] **노드 상태 색상** — 스텝을 이동하면 → 현재 노드(금색), 프론티어(보라색), 방문(초록색), 미방문(파란색)으로 구분된다
- [ ] **엣지 상태 색상** — 현재 탐색 엣지는 금색 실선, 방문된 엣지는 초록색 실선, 나머지는 파란색 점선으로 표시된다
- [ ] **방향 그래프 화살표** — 방향 그래프에서 → 엣지 끝에 화살표가 표시된다
- [ ] **엣지 가중치** — 가중치 그래프에서 → 엣지 중앙에 금색 숫자로 가중치가 표시된다
- [ ] **노드 드래그** — 노드를 마우스로 끌면 → 해당 노드의 위치가 이동되고 놓으면 다시 시뮬레이션에 참여한다
- [ ] **줌/팬** — 캔버스를 드래그하면 팬, 스크롤하면 0.5x~3x 범위에서 줌이 동작한다
- [ ] **범례** — 그래프 패널 좌측 하단에 → 노드/엣지 상태별 색상 범례가 표시된다

### 특수 자료구조
- [ ] **힙 트리 뷰** — HEAP으로 분류된 변수에서 → 이진 트리 형태의 SVG가 표시되고 노드에 배열 인덱스가 표시된다
- [ ] **큐 뷰** — QUEUE로 분류된 변수에서 → 좌측 DEQUEUE, 우측 ENQUEUE 화살표가 있는 파이프 형태로 표시된다
- [ ] **스택 뷰** — STACK으로 분류된 변수에서 → TOP 배지가 있는 세로 스택으로 표시되고 PUSH/POP 화살표가 표시된다
- [ ] **데크 뷰** — DEQUE로 분류된 변수에서 → 양쪽 끝이 열린 파이프 형태로 표시된다
- [ ] **Union-Find 뷰** — UNIONFIND로 분류된 변수에서 → 트리 포레스트가 표시되고 루트 노드에 "root" 레이블이 표시된다
- [ ] **방문 배열 뷰** — VISITED로 분류된 변수에서 → 방문된 셀에 초록색 체크, 미방문 셀에 "·"가 표시된다
- [ ] **거리 배열 뷰** — DISTANCE로 분류된 변수에서 → 히트맵 스타일로 거리 값에 따른 파란색 농도가 표시되고, 매우 큰 값은 "INF"로 표시된다

### 3D 볼륨 패널
- [ ] **슬라이스 뷰** — 3D 배열에서 → 축(fix z/fix x/fix y) 버튼으로 슬라이스 방향을 전환할 수 있다
- [ ] **포커스 슬라이스** — 현재 스텝에 해당하는 슬라이스에 → 파란색 테두리와 `[focus]` 레이블이 표시된다
- [ ] **셀 드래그 활성화** — 슬라이스 뷰에서 셀을 클릭/드래그하면 → 초록색으로 활성화되고, 활성 셀을 다시 클릭하면 비활성화된다
- [ ] **3D 뷰 전환** — SLICES/3D 토글을 클릭하면 → Three.js 기반 3D 볼륨 뷰로 전환된다
- [ ] **3D 네비게이션** — 3D 뷰에서 WASD 키와 마우스 드래그로 → 카메라를 이동/회전할 수 있다
- [ ] **확장 모드** — Expand 버튼을 클릭하면 → 전체 화면으로 확장되고 드래그 가능한 재생 컨트롤이 표시된다
- [ ] **한국어 키보드** — 3D 뷰에서 한국어 입력 상태(ㅈ/ㅁ/ㄴ/ㅇ)에서도 → WASD와 동일하게 네비게이션이 동작한다

## 엣지 케이스
- [ ] **힙 63개 초과** — 힙에 63개 이상 노드가 있으면 → "+N 노드 생략" 메시지가 표시된다
- [ ] **빈 그래프** — 그래프에 노드/엣지가 없으면 → "현재 단계에서는 그래프가 아직 구성되지 않았습니다" 메시지가 표시된다
- [ ] **Grid/Graph 토글** — 2D 데이터가 그리드와 그래프 모두 가능한 경우 → GRID/GRAPH 토글 버튼이 표시되어 전환할 수 있다
- [ ] **3D 24레이어 초과** — 3D 뷰에서 레이어가 24개를 초과하면 → 포커스 레이어 중심으로 24개 윈도우가 표시된다
- [ ] **방향 벡터 변수** — `dirs`, `delta` 같은 방향 벡터 변수는 → 그래프가 아닌 포맷팅된 텍스트로 표시된다

## 에러 케이스
- [ ] **런타임 에러 셀** — 에러가 발생한 스텝에서 현재 셀이 → 빨간색 테두리와 배경으로 표시되고 하단에 에러 메시지 배너가 나타난다
- [ ] **AI 실패 폴백** — AI 분석이 실패하면 → 변수 Key-Value 테이블이 표시되고 변경된 값은 주황색으로 강조된다
- [ ] **인식 불가 구조** — 시각화 가능한 자료구조가 없으면 → Key-Value 테이블이 기본 표시된다

## UI 상태
- [ ] **실행 전 빈 상태** — 실행 전 Grid/Linear 패널에 → 격자 아이콘과 "Visualization will appear here after execution" 메시지가 표시된다
- [ ] **실행 전 그래프** — 실행 전 Graph 패널에 → "실행 후 그래프가 표시됩니다" 메시지가 표시된다
- [ ] **시각화 헤더** — 분석 완료 후 → 패널 헤더에 "Graph Visualization", "Grid Visualization" 등 전략명이 표시된다
- [ ] **시간 복잡도 배지** — AI가 시간 복잡도를 분석하면 → 패널 헤더에 "T: O(...)" 배지가 표시된다
- [ ] **변수 변경 표시** — 변수 모니터에서 이전 스텝과 값이 달라진 변수에 → 주황색 마커와 배경이 표시된다
- [ ] **키 변수 강조** — AI가 핵심 변수로 지정한 변수는 → 변수 모니터에서 초록색 굵은 글씨로 표시된다
