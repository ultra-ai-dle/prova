# /review — Feature + 테스트 코드 품질 검사 + 커밋

## 목적
지정된 feature의 소스 코드와 테스트 코드를 품질 관점에서 검사하고, 개선 사항을 적용하여 커밋한다.

## 입력
- `$ARGUMENTS` — 검사 대상 feature 경로 또는 이름 (예: `execution/runtime`, `store/useProvaStore`)

## 실행 절차

1. **대상 파일 수집**: feature 소스 파일 + 관련 테스트 파일을 모두 읽는다.
2. **품질 검사 항목**:
   - **타입 안전성**: `any` 사용, 타입 단언(`as`) 남용, 누락된 타입 정의
   - **에러 핸들링**: try-catch 누락, 에러 무시(`catch(e) {}`), 에러 타입 미지정
   - **코드 구조**: 함수 길이 (50줄 초과 경고), 중복 로직, 불필요한 의존성
   - **네이밍**: 의미 불명확한 변수명, 일관성 없는 네이밍 컨벤션
   - **테스트 품질**: assertion 없는 테스트, 테스트 설명과 실제 검증 불일치, 커버리지 부족 영역
3. **개선 적용**: 발견된 이슈를 수정한다. 단, 동작 변경 없이 품질만 개선한다.
4. **검증**: type check(`npx tsc --noEmit`)를 실행하여 깨진 것이 없는지 확인한다.
5. **커밋**: `refactor: [feature명] 코드 품질 개선` 형식으로 커밋한다.

## 주의사항
- 기능 변경은 하지 않는다. 리팩토링 범위만 다룬다.
- 개선 사항이 없으면 커밋하지 않고 "이슈 없음" 으로 보고한다.
- 테스트가 없는 feature는 `/test`를 먼저 실행할 것을 권장한다.
