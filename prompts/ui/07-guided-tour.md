# Frogger — Screen 07: GuidedTour (가이드 투어)

## 목적

처음 방문한 사용자에게 Frogger 핵심 워크플로우(코드 작성 → 디버그 실행 → AI 시각화 → 단계별 탐색)를 7단계 투어로 안내한다.
기존 UI 위에 오버레이를 띄워 각 영역을 스포트라이트하며, 짧은 툴팁으로 역할을 설명한다.

---

## 전체 레이아웃

기존 Ready State(Screen 01) 위에 반투명 오버레이가 겹쳐진다.
현재 스텝의 대상 요소만 스포트라이트(밝게), 나머지는 어둡게 딤 처리.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    ██ 반투명 오버레이 (rgba 0,0,0,0.65) ██          │
│                                                                     │
│   ┌─ 스포트라이트 ──────────────┐                                   │
│   │  대상 요소 (밝게 노출)       │  ← clip-path 또는 SVG mask로 컷아웃 │
│   │  padding: 8px              │                                   │
│   └────────────────────────────┘                                   │
│         ▼                                                           │
│   ┌─ 툴팁 (max-width: 320px) ─────────────────────┐                │
│   │  1 / 7                              ━━━━━━░░░ │  ← 진행 바     │
│   │                                               │                │
│   │  코드 에디터                                    │  ← 제목 14px  │
│   │  여기에 알고리즘 코드를 작성하세요.                │  ← 본문 12px  │
│   │  BFS, DFS, DP 등 다양한 알고리즘을               │                │
│   │  지원합니다.                                    │                │
│   │                                               │                │
│   │  [건너뛰기]              [◀ 이전]  [다음 ▶]     │  ← 하단 버튼   │
│   └───────────────────────────────────────────────┘                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 투어 스텝 정의 (7단계)

사용자 여정 순서를 따른다: **소개 → 코드 입력 → 언어 → 실행 → 시각화 → 제어 → 변수 추적**.

| Step | 대상 요소                       | `data-tour`      | 툴팁 위치     | 제목                         | 본문                                                                                                                  |
| ---- | ------------------------------- | ---------------- | ------------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| 1    | 상단 헤더 바                    | `header`         | bottom-center | Prova에 오신 것을 환영합니다 | AI가 알고리즘의 실행 흐름을 시각화해 주는 디버거입니다. 핵심 기능을 안내해 드릴게요.                                  |
| 2    | Left Pane (코드 에디터)         | `editor`         | right         | 코드 에디터                  | 여기에 알고리즘 코드를 작성하세요. BFS, DFS, DP 등 다양한 알고리즘을 지원합니다. 기본 예시 코드가 미리 채워져 있어요. |
| 3    | 언어 선택기                     | `language`       | bottom-left   | 언어 선택                    | Python과 JavaScript를 지원합니다. 코드 패턴을 자동으로 감지하기도 해요.                                               |
| 4    | Input 영역 (stdin + 실행 버튼)  | `input`          | left          | 입력 & 실행                  | stdin 입력값을 작성하고 디버깅 시작 버튼을 클릭하세요. AI가 코드를 분석하고 시각화 전략을 결정합니다.                 |
| 5    | Center Pane (시각화 패널)       | `visualization`  | left          | 시각화 패널                  | AI가 선택한 전략(격자, 그래프, 선형 등)에 맞춰 알고리즘 동작을 애니메이션으로 보여줍니다.                             |
| 6    | 디버그 컨트롤 (슬라이더 + 버튼) | `debug-controls` | left          | 디버그 컨트롤                | 슬라이더를 드래그하거나 Prev/Next 버튼으로 단계별 이동하세요. Play로 자동 재생, 속도 조절도 가능합니다.               |
| 7    | 변수 모니터                     | `variables`      | left          | 변수 모니터                  | 각 단계에서 변수 값이 어떻게 변하는지 실시간으로 추적합니다. 변경된 변수는 노란색으로 강조됩니다.                     |

## 컴포넌트 상세

### 1. 오버레이 (Spotlight Overlay)

- 전체 화면 고정, `z-index: 60` (토스트 z-50 위).
- 배경: `rgba(0, 0, 0, 0.65)`.
- 대상 요소 주변을 CSS `clip-path: polygon()` 또는 SVG mask로 컷아웃.
- 컷아웃 패딩: 대상 요소로부터 `8px`.
- 스텝 전환 시 컷아웃 위치 트랜지션: `300ms ease-out`.

### 2. 툴팁 (Tour Tooltip)

- 배경: `#161b22`, 테두리: `1px solid #30363d`, `border-radius: 8px`.
- 그림자: `0 8px 24px rgba(0,0,0,0.4)`.
- 최대 너비: `320px`.
- 대상 요소를 가리키는 화살표(CSS triangle, 테두리 색상 일치).
- 위치: 대상 요소의 `getBoundingClientRect()` 기반 자동 계산, 뷰포트 밖으로 나가지 않도록 보정.

#### 툴팁 내부 구조

```
┌─ Tooltip ──────────────────────────────────┐
│  ━━━━━━━━━━━━━━░░░░░░░░░  ← 진행 바        │
│  1 / 7                    ← 스텝 표시 (10px)│
│                                            │
│  코드 에디터               ← 제목 (14px, #fff, bold) │
│  여기에 알고리즘 코드를      ← 본문 (12px, #8b949e)    │
│  작성하세요...                              │
│                                            │
│  건너뛰기         [◀ 이전]  [다음 ▶]         │
│  ↑ muted 텍스트    ↑ outline  ↑ #238636 채움 │
└────────────────────────────────────────────┘
```

### 3. 진행 표시기 (Progress Bar)

- 툴팁 상단에 가는 바(height: `2px`).
- 배경: `#30363d` (트랙), 채움: `#3fb950` (진행).
- 너비: `(currentStep / totalSteps) * 100%`.

### 4. 버튼 그룹

- **건너뛰기**: 좌측, 텍스트 버튼, 색상 `#8b949e`. 호버 시 `#c9d1d9`.
- **이전**: outline 버튼, 테두리 `#30363d`, 텍스트 `#c9d1d9`. Step 1에서는 숨김.
- **다음**: 채움 버튼, 배경 `#238636`, 텍스트 `#fff`. 호버 시 `#2ea043`.
- **마지막 스텝**: "다음" 대신 "시작하기!" 표시, 같은 스타일.

## 상태 변형

### 트리거 조건

| 조건                                      | 동작                                                               |
| ----------------------------------------- | ------------------------------------------------------------------ |
| 첫 방문 (`prova:tourCompleted` 없음)      | `uiMode === "ready"` && `pyodideStatus === "ready"` 되면 자동 시작 |
| 재방문 (`prova:tourCompleted === "true"`) | 투어 시작하지 않음                                                 |
| 설정 아이콘 클릭 → "가이드 투어 다시보기" | 수동 시작                                                          |

### 중단 / 재개

| 이벤트                                       | 동작                                                |
| -------------------------------------------- | --------------------------------------------------- |
| "건너뛰기" 클릭 또는 `Escape` 키             | 투어 즉시 종료, `prova:tourCompleted = "true"` 저장 |
| 마지막 스텝 "시작하기!" 클릭                 | 투어 완료, `prova:tourCompleted = "true"` 저장      |
| 디버그 실행 버튼 클릭 (투어 중)              | 투어 자동 종료, completed 처리                      |
| `pyodideStatus` → `"loading"` 또는 `"error"` | 투어 일시정지, ready 복귀 시 현재 스텝에서 재개     |
| 브라우저 리사이즈                            | 스포트라이트 + 툴팁 위치 재계산                     |

### 키보드 지원

| 키               | 동작          |
| ---------------- | ------------- |
| `Escape`         | 투어 건너뛰기 |
| `→` 또는 `Enter` | 다음 스텝     |
| `←`              | 이전 스텝     |

| 대상                | 동작                                    | 시간             |
| ------------------- | --------------------------------------- | ---------------- |
| 스포트라이트 컷아웃 | 이전 위치 → 다음 위치 이동              | `300ms ease-out` |

| 진행 바             | 너비 전환                               | `300ms ease-out` |

## 접근성

- 툴팁: `role="dialog"`, `aria-modal="true"`, 스텝별 `aria-label`.
- 포커스 트랩: 툴팁 내부 버튼(건너뛰기 / 이전 / 다음)에 포커스 순환.
- 대상 요소에 `aria-describedby`로 툴팁 연결.

## UX 포인트

- 7스텝은 1분 이내에 끝나는 분량. 스텝이 너무 많으면 이탈률이 높으므로 7개 이하 유지.
- 첫 스텝(환영)에서 "이 투어는 약 1분 정도 걸립니다" 같은 시간 안내를 넣으면 완주율이 올라감.
- 스포트라이트 영역 바깥 클릭은 무시(다음 스텝 이동 아님) — 실수로 투어를 잃는 것을 방지.
- "건너뛰기"는 항상 보이되 시각적으로 약하게(muted) 처리하여 완주를 유도.
- 마지막 스텝의 "시작하기!" 버튼은 사용자에게 능동적 종료 경험을 줌.
