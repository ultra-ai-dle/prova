# Prova — Screen 03: Visualizing (메인 시각화 — 정상 완료)

## 목적
AI 분석 및 청크 스트리밍이 모두 완료된 후, 사용자가 step-by-step으로 알고리즘을
탐색하는 **핵심 인터랙션 화면**. 이 화면이 Prova의 메인 경험이다.

데모 시나리오: BFS 격자 탐색 (GRID + LINEAR HYBRID 전략 기준으로 기술).

---

## 컴포넌트 상세

### 1. 상단 헤더 배지 (알고리즘 감지 완료)
- pill 모양, 배경 진한 녹색(`#1a4731`), 테두리 `#3fb950`.
- 좌측 작은 녹색 채워진 원(●) + 텍스트: "BFS — 격자 탐색" (흰색).
- Phase 1의 `display_name` 값을 그대로 표시.

---

### 2. Left Pane — Monaco 에디터

- **현재 step 줄 하이라이트**: 노란색(`#f0e68c` 계열) 배경, 왼쪽 가는 황색 세로선.
- 줄 번호 옆 **Scope Breadcrumb 인디케이터**: 재귀/함수 호출 중이면 해당 줄 번호 배경을 달리 표시.
- 에디터 편집 가능(실행 완료 후 코드 수정 허용). 수정 시 "다시 실행" 유도 배너 표시.

**Breadcrumb (에디터 상단 또는 Right Pane 상단 — 함수 스코프 표시):**
```
[ <global> > bfs (depth: 1) ]
```
- 작은 pill 체인 형태. 현재 활성 함수는 밝게, 부모는 희미하게.
- `scope.func` + `scope.depth` 값 사용.

---

### 3. Center Pane — 시각화 패널

#### GRID 패널 (주 시각화, HYBRID에서 상단 차지)
```
     0    1    2    3    4
  ┌────┬────┬────┬────┬────┐
0 │    │ ██ │    │    │    │  ← ██ = 벽 (어두운 회색)
  ├────┼────┼────┼────┼────┤
1 │ ✓  │ →  │    │    │    │  ← ✓ = 방문 완료 (청록 배경)  → = 현재 (밝은 청록)
  ├────┼────┼────┼────┼────┤
2 │    │    │    │    │    │
  └────┴────┴────┴────┴────┘
```
- **셀 상태별 색상**:
  | 상태 | 배경 | 아이콘 |
  |---|---|---|
  | 미방문(기본) | `#1c2128` (어두운) | 없음 |
  | 방문 완료 | `#0d4429` (진녹) | 작은 체크 또는 없음 |
  | 현재 탐색 | `#3fb950` (밝은 녹) | 강한 테두리 |
  | 경로 표시 | `#1f6feb` (파란) | 없음 |
  | 벽 | `#30363d` (진회색) | 없음 |
  | 에러 | `#da3633` (빨간) | × |
- 셀 전환: `markVisited`, `focus` 액션에 따라 200ms fade 또는 scale 애니메이션.
- 셀 내부 폰트: monospace, 소문자 좌표 또는 값.

#### LINEAR 패널 (서브, HYBRID에서 하단 차지)
큐(queue)를 수평 리스트로 표시:
```
  HEAD  →  [ (1,0) ]  [ (0,1) ]  [ (2,1) ]  →  TAIL
```
- 각 항목: 둥근 사각형 칩, 배경 `#1f3555`, 테두리 `#388bfd`.
- `push` 액션: 오른쪽에서 슬라이드인 + scale-up.
- `pop` 액션: 왼쪽 항목 scale-down + 사라짐.
- 수평 스크롤(항목 많을 시 overflow-x).

### 4. Right Pane

#### A. 스코프 Breadcrumb (최상단)
```
[ <global> ] › [ bfs ] (depth: 1)
```
- 회색 소문자, 가볍게.
- 재귀가 없는 일반 호출: "main › bfs" 형태.
- depth 1이상: "bfs (depth: 2)" 추가 표시.

#### B. 변수 모니터
- `key_vars` + `var_mapping`에 의해 **핵심 변수가 상단 강조**:
  - `queue` (MAIN_QUEUE 역할): 배지 스타일 레이블 + 값 (deque 형태로 렌더링).
  - `visited` (2D 리스트): 미니 그리드 썸네일 또는 "2D list [5×5]" 텍스트.
  - `dist`, `i`, `j` 등 스칼라: 단순 Key-Value 행.
- **변경된 변수**: 이전 step 대비 값이 달라지면 **노란색 배경 highlight 0.5초 후 페이드아웃**.
- 부모 프레임 변수(`parent_frames`):
  - 구분선 아래 "외부 스코프 (depth: 0)" 레이블.
  - opacity 35%, 이탤릭 또는 흐릿한 색상.

```
┌─ 변수 모니터 ────────────────┐
│  📌 KEY VARS                  │
│  queue  [ (1,0), (0,1) ]      │  ← 강조 배지
│  visited  2D list [5×5] [▸]   │  ← 펼치기 토글
│  ─────────────────────────── │
│  dist   3                     │
│  i      1                     │
│  j      0                     │
│  ─────────────────────────── │
│  외부 스코프 (depth: 0)        │
│  grid   2D list [5×5] (희미)  │
└──────────────────────────────┘
```

#### C. AI 설명 카드 (현재 step)
```
┌─ 현재 단계 ──────────────────┐
│  L.15  "큐에서 (1,0) 꺼내기" │  ← 줄 번호 + 설명 텍스트
│                               │
│  visual_actions:              │
│   · focus GRID (1,0)          │  ← 참고용 action 미리보기(선택)
│   · pop LINEAR                │
└──────────────────────────────┘
```
- 카드 배경: `#161b22`.
- 줄 번호: 소문자 회색 pill.
- 설명 텍스트: 흰색, 16px, line-height 1.5.

#### D. 콘솔 출력
- `print()` 결과가 있으면 하단 접이식 패널:
  ```
  ▸ 콘솔  "BFS 시작: (0,0)"  "방문: (1,0)"
  ```
- 기본 접힌 상태(collapsed). 새 출력 있으면 뱃지 숫자 증가.

### 5. Input Panel (실행 완료 상태)
- stdin textarea: 이전에 입력한 값 표시 (읽기 전용).
- 실행 버튼: "▶ 다시 실행" 텍스트로 변경 (활성화, 클릭 시 처음부터 재실행).

### 6. Timeline Bar (핵심 인터랙션)

```
  [시작]──●──◆──●──●──◆──[❚현재]──●──◆──●──[끝]     ◀  ▶  ▷  ×1▾
```

**Dot 종류 및 색상**:
| Dot | 의미 | 모양 | 색상 |
|---|---|---|---|
| ● 일반 | 일반 실행 줄 | 작은 원 (6px) | `#30363d` 회색 |
| ◆ 분기 | if/else 조건 줄 | 다이아몬드 | `#8b5cf6` 보라 |
| ◉ 루프 | for/while 줄 | 원 + 속채움 | `#1d4ed8` 파랑 |
| ❚ 현재 | 현재 step | 큰 흰색 원 | `#ffffff` 흰색 |

**현재 step 마커**: 큰 흰색 원, 타임라인 트랙 위에 올라탐. 드래그로 step 이동 가능.

**재생 컨트롤**:
- `◀` 이전 step (클릭 또는 키보드 ← 키).
- `▶` 다음 step (클릭 또는 키보드 → 키).
- `▷` / `⏸` Auto-play 토글.
- 속도 드롭다운: `×0.5` / `×1` / `×1.5` / `×2`.

## UX 포인트
- **노란 에디터 하이라이트 + 타임라인 마커 + Center 셀 활성화**가 동시에 반응해 step 이동이 직관적.
- 변수 변경 시 yellow flash는 "무엇이 바뀌었는지"를 명시적으로 알려주는 핵심 UX.
- 타임라인 드래그로 특정 step으로 점프하면 에디터/시각화/변수 패널이 전부 해당 step 상태로 즉시 전환.
- Breadcrumb는 작게 있지만, 재귀 학습 시 "지금 어디에 있는가"를 알 수 있는 중요한 나침반.
- 콘솔 출력 패널은 기본 접어두어 시각화 공간을 최대 확보.
