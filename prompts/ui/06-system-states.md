# Prova — Screen 06: System States (시스템 상태)

## 목적
개별 step 에러(04번 화면)가 아닌 **시스템 수준의 장애** 상태 모음.
각 상태는 `pyodideStatus` 값으로 구분된다.

| `pyodideStatus` | 원인 | 화면 |
|---|---|---|
| `loading` | 페이지 진입, Pyodide 다운로드 중 | 01번 화면 변형 (배너 추가) |
| `reinitializing` | 실행 5초 초과 → worker 강제 종료 후 재로드 | 토스트 + 스피너 |
| `error` | Pyodide 로드 실패 (치명적) | 전체 에러 배너 |

---

## State A: `loading` — Python 환경 준비 중

Ready State(01번)와 동일한 레이아웃에 상단 배너 하나 추가.

```
┌──── 안내 배너 (height: 36px, amber 배경) ────────────────────────────────────┐
│  ⏳  Python 환경 준비 중입니다. 잠시만 기다려 주세요.                        │
└─────────────────────────────────────────────────────────────────────────────┘
```

- 배경: `#3d2b00`, 텍스트: `#e3b341`, 왼쪽에 작은 회전 스피너 아이콘.
- 실행 버튼: **비활성화** (회색, disabled).
- 배너는 `pyodideStatus: ready` 신호 수신 시 fade-out으로 사라짐.
- 에디터와 나머지 UI는 모두 인터랙션 가능 (코드 입력, stdin 입력 허용).

**UX 포인트:**
- 배너가 전체를 차단하지 않으므로 사용자는 기다리는 동안 코드를 미리 작성할 수 있음.
- 준비 완료 시 배너가 사라지며 실행 버튼이 활성화되는 흐름이 자연스러운 진입 경험을 만듦.

---

## State B: `reinitializing` — Timeout 후 환경 재초기화

실행 5초 초과 시 발생. worker를 종료하고 새 Pyodide를 로드하는 중.

### 토스트 알림 (우측 하단)

#### Step 1: Timeout 발생 직후
```
┌──── 토스트 (width: 380px, 우측 하단) ────────────────────┐
│  ⚠️  실행 시간이 너무 길어 안전을 위해 실행을 중단했습니다.│
│       환경을 재설정합니다.                                 │
└───────────────────────────────────────────────────────────┘
```
- 배경: `#5a3600`, 테두리 `#e3b341` (amber).
- 자동으로 사라지지 않음 — `reinitializing` 완료까지 유지.
- 우상단 닫기(×) 없음 — 상태 진행 중임을 알리는 목적.

#### Step 2: 재초기화 완료
기존 토스트가 사라지고 새 토스트 등장:
```
┌──── 토스트 ─────────────────────────────────────────────┐
│  ✅  환경 준비 완료. 코드를 수정 후 다시 시도해 주세요.  │
└─────────────────────────────────────────────────────────┘
```
- 배경: `#0d4429`, 테두리 `#3fb950` (녹색).
- 4초 후 자동 fade-out.

### 실행 버튼 상태 변화

| 단계 | 버튼 상태 |
|---|---|
| Timeout 발생 전 (실행 중) | "실행 중..." 회색, 비활성화 |
| Timeout 발생 직후 | **"🔄 초기화 중..."** 회전 스피너 + 회색, 비활성화 |
| 재초기화 완료 | **"▶ 실행"** 활성화 (녹색) |

```
[ 🔄 초기화 중... ]  ← 버튼 UI 예시
```
- 텍스트 좌측에 인라인 회전 아이콘(CSS animation, 1회전/초).
- 배경: `#21262d` (어두운 회색), 클릭 불가.

### 배경 UI 상태
- 에디터, 타임라인, 변수 패널 등 나머지 UI: **정상 표시 유지** (차단하지 않음).
- 이미 완료된 trace 결과(rawTrace)가 있다면 유지됨.

**UX 포인트:**
- 버튼의 텍스트 변화가 "뭔가 진행 중"임을 명확히 전달 — 사용자가 반복 클릭 시도 방지.
- 재초기화 완료 토스트에 "코드를 수정 후 다시 시도"를 명시해 무한 루프 코드를 고치도록 안내.

---

## State C: `error` — Pyodide 로드 치명적 실패

Pyodide 바이너리 다운로드 실패 등 복구 불가 상태.

```
┌──── 에러 배너 (height: 48px, 빨간 배경, 화면 전체 너비) ────────────────────┐
│  ✕  Python 환경 초기화에 실패했습니다. 페이지를 새로고침해 주세요.  [새로고침]│
└─────────────────────────────────────────────────────────────────────────────┘
```

- 배경: `#5a1212`, 테두리 `#da3633` (빨간).
- 텍스트: "Python 환경 초기화에 실패했습니다. 페이지를 새로고침해 주세요."
- 우측: **[새로고침]** 버튼 (흰색 아웃라인) — 클릭 시 `window.location.reload()`.
- 배너는 닫기 불가 (치명적 상태임을 인식).

### 실행 버튼 상태
- 비활성화, 텍스트 "실행 불가".
- 에디터와 나머지 UI는 읽기 전용으로 유지 (코드 손실 방지).

**UX 포인트:**
- 배너가 화면 상단에 고정되어 사용자가 스크롤해도 항상 보임.
- [새로고침] 버튼을 배너에 직접 배치해 해결 동작까지의 클릭 수를 최소화.
- 에디터의 코드는 남아있어 새로고침 후에도 localStorage 복구가 가능하도록 설계 여지 남김 (향후).

---

## 상태별 실행 버튼 스펙 요약

| `pyodideStatus` | 버튼 텍스트 | 색상 | 클릭 가능 |
|---|---|---|---|
| `loading` | ▶ 실행 | 회색 (disabled) | ✕ |
| `ready` | ▶ 실행 | 녹색 (활성) | ✓ |
| `running` | 실행 중... | 회색 (disabled) | ✕ |
| `reinitializing` | 🔄 초기화 중... | 회색 + 스피너 | ✕ |
| `error` | 실행 불가 | 빨간 (disabled) | ✕ |

---

## 토스트 시스템 공통 스펙

| 유형 | 색상 | 자동 닫힘 | 위치 |
|---|---|---|---|
| 성공 (✅) | 녹색 (`#0d4429` / `#3fb950`) | 4초 | 우측 하단 |
| 경고 (⚠️) | Amber (`#5a3600` / `#e3b341`) | 수동 | 우측 하단 |
| 에러 (✕) | 빨간 (`#5a1212` / `#da3633`) | 수동 | 우측 하단 |

- 토스트 너비: 최대 400px, `border-radius: 8px`.
- 등장: 오른쪽에서 슬라이드인 (200ms ease-out).
- 사라짐: fade-out (300ms).
- 동시에 최대 3개 스택 가능 (오래된 것부터 위에 위치).
