# Prova — Screen 01: Ready State (준비 상태)

## 목적
사용자가 처음 진입하거나 이전 실행이 끝난 뒤의 기본 상태.
Python 환경이 백그라운드에서 로드 완료되어 실행 버튼이 활성화된 상태다.

---

## 전체 레이아웃

전체 화면 3열 고정 레이아웃. 배경은 어두운 에디터 테마(dark, 거의 검정에 가까운 #0d1117 계열).
열 비율은 Left 30% : Center 45% : Right 25%.

```
┌──── 상단 헤더 바 (height: 48px) ────────────────────────────────────────────┐
│  Prova  로고 (좌)   │   [알고리즘 감지 배지 — 실행 전은 빈 상태]   │  도움말 아이콘 (우)  │
└─────────────────────────────────────────────────────────────────────────────┘
┌─────────────────┬──────────────────────────┬──────────────────────┐
│  Left Pane      │  Center Pane             │  Right Pane          │
│  (30%)          │  (45%)                   │  (25%)               │
│                 │                          │                      │
│  Monaco Editor  │  [Placeholder]           │  [변수 모니터 빈 상태] │
│                 │  "코드를 실행하면         │                      │
│  지원 패키지 안내  │   시각화가 표시됩니다"   │  [AI 설명 카드 빈 상태]│
│  배지 (하단 고정) │  희미한 그리드 격자 윤곽  │                      │
└─────────────────┴──────────────────────────┴──────────────────────┘
┌──── Input Panel (height: 80px) ─────────────────────────────────────────────┐
│  stdin:  [ textarea placeholder "줄마다 입력값을 작성하세요" ]   [ ▶ 실행 ]  │
└─────────────────────────────────────────────────────────────────────────────┘
┌──── Timeline Bar (height: 48px) ────────────────────────────────────────────┐
│  [빈 타임라인 — 점 없음]                      ◀  ▶  ▷  ×0.5  ×1  ×2         │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 컴포넌트 상세

### 1. 상단 헤더 바
- **Prova 로고**: 흰색 텍스트, 좌측 배치. "Prova"라는 단어에 강조색(예: 보라 또는 청록) 액센트.
- **알고리즘 감지 배지**: 헤더 중앙. 실행 전이라 내용 없음 → 연한 회색 점선 경계선에 "알고리즘 감지 전" 회색 텍스트. 작은 사이즈(pill shape, 24px height).
- **도움말 아이콘**: 우측. `?` 아이콘.

### 2. Left Pane — Monaco 에디터
- 배경: `#161b22` 또는 VS Code Dark 계열.
- 예시 코드(BFS)가 미리 로드된 상태로 표시. 줄 번호 표시.
- 에디터 하단 고정 배지:
  ```
  Python 3.11 표준 라이브러리 지원  ·  외부 패키지(numpy, networkx 등) 미지원
  ```
  작은 폰트(11px), 회색 배경 pill.

### 3. Center Pane — Placeholder
- 배경: 에디터보다 살짝 밝은 어두운 회색(`#1c2128`).
- 중앙에 아이콘(격자 또는 플레이 버튼 실루엣) + 텍스트:
  - 큰 텍스트: "코드를 실행하면 시각화가 여기에 표시됩니다"
  - 작은 서브텍스트(회색): "BFS, DFS, 스택, 큐 등의 알고리즘을 자동으로 감지합니다"
- 희미한 격자 dot-pattern 배경(opacity 5~10%).

### 4. Right Pane — 변수 모니터 + AI 설명
- 상단 섹션 레이블: "변수 모니터" (소문자, 회색 상단 캡션).
  - 내용 없음 → 회색 빈 박스에 "실행 후 변수가 표시됩니다" 텍스트.
- 하단 섹션 레이블: "AI 단계 설명".
  - 내용 없음 → 동일 스타일 빈 상태.
- 두 섹션은 가는 구분선(1px, `#30363d`)으로 나뉨.

### 5. Input Panel
- 배경: `#13161a`.
- **stdin textarea**: 왼쪽 영역(약 75% 너비). 단색 어두운 배경, 가는 테두리, placeholder 텍스트 "줄마다 입력값을 작성하세요 (예: 5 3↵2 4 1)". 높이는 2~3줄 표시 정도.
- **실행 버튼**: 오른쪽. 강조색 채워진 버튼(예: `#238636` 녹색 또는 브랜드 청록). 텍스트 "▶ 실행". 호버 시 밝아짐. 활성 상태(enabled).
- 버튼 왼쪽 아래에 매우 작은 회색 텍스트: "배열 최대 20 · 노드 최대 15개".

### 6. Timeline Bar
- 배경: `#0d1117`.
- 빈 타임라인 트랙(점 없음, 가는 회색 선만).
- 우측: 재생 컨트롤 아이콘 그룹. **비활성 상태(disabled, opacity 40%)** — 실행 전.
  - `◀` 이전 `▶` 다음 `▷` 재생/일시정지, 속도 칩셋(×0.5 / ×1 / ×2).

---

## 상태 변형

| 상태 | 변화 |
|---|---|
| `pyodideStatus: loading` | 실행 버튼 비활성화 + 상단에 얇은 배너 "Python 환경 준비 중..." (amber 색, 배너 높이 32px) |
| `pyodideStatus: ready` | 이 화면이 기본 상태. 실행 버튼 활성화. 배너 없음. |

---

## UX 포인트
- 에디터에 기본 예시 코드(BFS 5줄 정도)를 미리 채워두면 사용자가 "무엇을 넣어야 하는지" 즉시 이해함.
- Center placeholder의 아이콘은 알고리즘 시각화 실루엣(격자, 노드-엣지)으로 시각적 기대감 형성.
- 환경 로딩 배너는 조용하게(amber 계열, 방해 없이) 상태를 알림.
