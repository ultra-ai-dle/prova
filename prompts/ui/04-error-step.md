# Prova — Screen 04: Error Step (에러 Step Break 상태)

## 목적
코드 실행 중 `IndexError`, `KeyError` 등 런타임 에러가 발생한 step에서
**자동 Break**된 상태. 사용자가 에러 원인을 파악하고 Rewind하여 이전 step을
역추적할 수 있는 상태다.

---

## 전체 레이아웃

```
┌──── 상단 헤더 바 ────────────────────────────────────────────────────────────┐
│  Prova 로고  │  [● BFS — 격자 탐색]  │  ⚠️ IndexError at L.17  │  도움말   │
└─────────────────────────────────────────────────────────────────────────────┘
┌─────────────────┬──────────────────────────┬──────────────────────┐
│  Left Pane      │  Center Pane             │  Right Pane          │
│                 │                          │                      │
│  Monaco Editor  │  GRID 시각화             │  ── 변수 모니터 ──   │
│  에러 줄        │  에러 셀 빨간 X 표시     │  (에러 step 기준)    │
│  빨간 하이라이트 │                          │                      │
│  + 에러 마커    │                          │  ── 에러 분석 ──    │
│                 │                          │  runtimeError 메시지 │
│                 │                          │  aiError 설명 카드   │
│                 │                          │  fix_hint            │
└─────────────────┴──────────────────────────┴──────────────────────┘
┌──── Input Panel ────────────────────────────────────────────────────────────┐
│  [에러 안내 배너]  코드 수정 후 다시 실행해 보세요.   [ ▶ 다시 실행 ]      │
└─────────────────────────────────────────────────────────────────────────────┘
┌──── Timeline Bar ───────────────────────────────────────────────────────────┐
│  ──●──◆──●──●──◆──●──[✕ 에러]     ◀  ▶  ▷  ×1▾                           │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 컴포넌트 상세

### 1. 상단 헤더 에러 알림
- 알고리즘 배지 우측에 추가 에러 인디케이터:
  ```
  ⚠️ IndexError at L.17
  ```
  - 배경: 진한 빨간(`#5a1a1a`), 테두리 `#da3633`.
  - 클릭 시 Right Pane 에러 카드로 포커스 스크롤.

---

### 2. Left Pane — 에러 줄 표시

- **에러 줄 배경**: 빨간(`#3d0b0b` 등 어두운 빨간).
- **왼쪽 에러 마커**: 에디터 gutter에 빨간 ● 아이콘.
- **인라인 에러 메시지**: 에러 줄 끝 또는 다음 줄에 인라인으로 표시:
  ```python
  17  grid[i][j] = 1  ← ← ← IndexError: list index out of range
  ```
  Monaco의 `editor.setModelMarkers()` 스타일 — 빨간 밑줄 + hover 툴팁.
- 이전 step들의 줄들은 정상 상태(회색 또는 연녹색) 유지.

---

### 3. Center Pane — 에러 셀 시각화

```
     0    1    2    3    4
  ┌────┬────┬────┬────┬────┐
0 │    │ ██ │    │    │    │
  ├────┼────┼────┼────┼────┤
1 │ ✓  │ ✓  │    │    │    │
  ├────┼────┼────┼────┼────┤
... (중략)
5 │ ✕  │    │    │    │    │  ← 범위 초과 셀 (row 5는 존재하지 않음)
  └────┴────┴────┴────┴────┘
```

- **에러 셀**: 빨간 배경(`#5a1212`) + 중앙에 흰색 `✕` 아이콘.
  - `markError` visual_action에 의해 표시.
  - 2초 간격으로 pulse (빨간 테두리 glowing 효과) — 시선 유도.
- 에러 이전까지 방문한 셀들은 정상 방문 완료 상태로 유지.

---

### 4. Right Pane

#### A. 변수 모니터 (에러 step 기준)
- 에러를 유발한 변수들 강조:
  - `i = 5` (범위 초과 인덱스) → 빨간 텍스트 또는 빨간 배경.
  - `grid` (5×5) → "행 최대 인덱스: 4" 힌트 표시.
- 그 외 변수는 정상 표시.

#### B. 에러 분석 패널 (핵심)

```
┌─ ⚠️ 에러 발생 ─────────────────┐
│  IndexError                     │  ← runtimeError.type
│  list index out of range        │  ← runtimeError.message
│  L.17                           │  ← 에러 발생 줄
│─────────────────────────────── │
│  🔍 AI 원인 분석                │
│                                 │
│  원인:                          │
│  grid의 행 크기(5)를 초과하는   │  ← aiError.root_cause
│  인덱스(5)에 접근함             │
│                                 │
│  수정 힌트:                     │
│  탐색 전 행/열 범위 조건        │  ← aiError.fix_hint
│  (0 <= r < rows)을              │
│  먼저 확인하세요                │
│─────────────────────────────── │
│  [ ↩ 에러 이전으로 되돌아가기 ] │  ← Rewind CTA 버튼
└────────────────────────────────┘
```

- 에러 패널 테두리: 빨간 (`#da3633`), 배경: `#1a0a0a`.
- 구분선으로 `runtimeError`(Pyodide 팩트)와 `aiError`(AI 해석)를 명확히 분리.
- **"에러 이전으로 되돌아가기" 버튼**: 클릭 시 에러 step 바로 이전 step으로 이동. Rewind의 진입점.
- `aiError`가 null이면 AI 원인 분석 섹션 숨김.

#### C. 현재 step AI 설명 카드
- 에러 step에서도 `explanation`이 있으면 표시:
  ```
  L.17  "인덱스 범위 초과 — 탐색 실패"
  ```
- 배경: 빨간 계열 tint (`#2d0b0b`).

---

### 5. Input Panel (에러 상태)
- 상단에 연한 빨간 배너 (height 36px):
  ```
  ⚠️  런타임 에러가 발생했습니다. 코드를 수정 후 다시 실행해 보세요.
  ```
- 실행 버튼: "▶ 다시 실행" (활성화). 에러가 있어도 재실행 가능.

---

### 6. Timeline Bar (에러 마커)

```
  [시작] ──●──◆──●──●──◆──●── [✕] ← 에러
                                │
                                현재 위치 (자동 Break)
```

- **에러 Dot**: 빨간 배경 + `✕` 흰색. 크기 10px (다른 Dot보다 큰 강조).
- 에러 Dot 위치에 현재 마커 자동 이동.
- 재생 컨트롤: 활성화. `◀` 이전 step으로 Rewind 가능.
- 에러 Dot 이후로는 이동 불가 (마지막 step이 에러임).

---

## Rewind 인터랙션 흐름

1. 사용자가 `◀` 버튼 또는 Timeline 드래그로 이전 step 이동.
2. 에디터 하이라이트 → 이전 줄(빨간→노란)으로 이동.
3. 변수 모니터 → 이전 step의 `vars` 상태로 롤백.
4. Grid 셀 → 방문 완료 셀이 미방문으로 점진적 해제.
5. Right Pane 에러 패널 → 에러 step이 아니면 숨겨짐(일반 AI 설명 카드로 교체).

---

## UX 포인트
- **에러 발생 즉시 자동 Break** — 사용자가 "어디서 터졌지?"를 찾을 필요 없음.
- `runtimeError`(팩트)와 `aiError`(해석)를 패널에서 물리적으로 분리해 신뢰도 구분.
- "에러 이전으로 되돌아가기" CTA로 Rewind를 모르는 사용자에게 역추적 동작을 명시적으로 안내.
- Timeline의 빨간 ✕ Dot가 에러 위치를 한눈에 파악하게 해주는 앵커 역할.
- 에러 상황에서도 다른 step을 자유롭게 탐색할 수 있어 "에러 직전 변수 흐름"을 분석 가능.
