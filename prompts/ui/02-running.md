# Prova — Screen 02: Running State (실행 중 / AI 분석 로딩)

## 목적
사용자가 실행 버튼을 클릭한 후, Pyodide가 코드를 실행하고
AI가 Phase 1 분석 → Phase 2 청크 스트리밍을 완료하기 전까지의 상태.

크게 두 단계로 나뉘지만 **같은 화면을 점진적으로 채우는 방식**으로 표현한다:
1. Pyodide 실행 중 (rawTrace 수집)
2. AI 청크 스트리밍 중 (설명이 실시간으로 채워짐)

---

## 전체 레이아웃

Ready State와 동일한 3열 구조. 추가로 상단에 프로그레스 바가 붙는다.

```
┌──── 상단 프로그레스 바 (height: 3px, 화면 전체 너비, animated) ──────────────┐

└─────────────────────────────────────────────────────────────────────────────┘
┌──── 상단 헤더 바 ────────────────────────────────────────────────────────────┐
│  Prova 로고  │  [알고리즘 분석 중...] 스피너 배지  │  도움말                 │
└─────────────────────────────────────────────────────────────────────────────┘
┌─────────────────┬──────────────────────────┬──────────────────────┐
│  Left Pane      │  Center Pane             │  Right Pane          │
│                 │                          │                      │
│  Monaco Editor  │  로딩 상태 (Phase 1)     │  변수 모니터          │
```

## 컴포넌트 상세

### 1. 상단 프로그레스 바
- 화면 최상단, 3px 높이의 얇은 바.
- 색상: 브랜드 청록(`#58a6ff` 또는 `#3fb950`).
- 무한 shimmer 애니메이션(좌→우 반짝임) — 진행률을 알 수 없으므로 indeterminate 스타일.
- Phase 1 완료 → 50% 고정, Phase 2 청크가 쌓일수록 50→100%.

### 2. 헤더 배지 (실행 중 상태)
- 텍스트: "알고리즘 분석 중..."
- 좌측에 작은 회전 스피너 아이콘.
- 배지 배경: 연한 amber(주황 계열) — 진행 중임을 시각적으로 암시.

### 3. Left Pane — 에디터 (실시간 하이라이트)
- Pyodide 실행 중: rawTrace 각 step의 `line` 번호에 맞춰 **연두색(`#3fb950`) 줄 배경 하이라이트** 이동.
  - 매 step마다 새 줄에 부드럽게 이동(200ms transition).
- 에디터 자체는 읽기 전용(편집 불가) — 실행 중 코드 수정 방지.

### 4. Center Pane

#### Phase 1 완료 전 (rawTrace 아직 없음)
- 배경에 부드러운 pulse 애니메이션.
- 텍스트: "코드 실행 중..." (중앙 정렬, 회색).
- 아래 서브텍스트: "변수 상태를 수집하고 있습니다".

#### Phase 1 완료 후 (rawTrace + branchLines 도착)
- **타임라인 Dot이 즉시 표시됨** (Center Pane 하단 또는 Timeline Bar에).
- Center Pane 본문: AI 시각화 전략 결정 대기 → 희미한 격자/그래프 실루엣 pulse.
- 우상단 작은 텍스트: "실행 완료 · AI 해석 중..." (회색 소문자).

#### Phase 2 스트리밍 중
- **그리드/리스트 셀이 AI 청크에 맞춰 하나씩 활성화**.
  - 아직 설명이 없는 step의 셀: 회색 골격(skeleton) 상태.
  - 청크 도착 시 해당 step 셀이 활성색으로 전환 (fade-in 200ms).

### 5. Right Pane

#### 변수 모니터
- rawTrace 도착 즉시: 현재 step의 `vars`를 Key-Value로 표시.
- 변수 카드: `변수명` (좌측 회색 레이블) + `값` (우측 흰색).
- 아직 AI key_vars 결정 전이므로 강조 없이 평등하게 나열.

#### AI 설명 카드
- Phase 2 청크 도착 전: 스켈레톤 카드 3~5개 (shimmer 배경의 빈 줄).
- 청크 도착 시: 위에서부터 순서대로 실제 텍스트로 전환 (fade-in).
- 각 카드: 아주 작은 줄 번호(`L.12`) + 설명 텍스트(한국어, 한 줄).

### 6. Input Panel
- stdin textarea: 편집 불가 처리(blur 또는 pointer-events: none).
- 실행 버튼: **비활성화 + 텍스트 변경**: "실행 중..." + 왼쪽에 작은 회전 아이콘.
  - 색상: 밝은 녹색 → 회색으로 변경.

### 7. Timeline Bar
- rawTrace 도착 후: Dot들이 순서대로 표시됨.
  - 루프 줄(`branchLines.loop`): 파란 원 Dot.
  - 분기 줄(`branchLines.branch`): 연한 보라 다이아몬드 Dot.
  - 일반 줄: 작은 회색 원.
- 현재 실행 위치 표시: 밝은 흰색 원형 마커가 Dot 위를 이동.
- 재생 컨트롤: 여전히 **비활성화(disabled)** — 모든 청크 완료 전.

## UX 포인트
- **진행 중임을 여러 레이어에서 동시에 피드백**: 프로그레스 바 + 에디터 줄 이동 + 타임라인 Dot 증가.
- **에디터 줄 하이라이트** 이동이 가장 강렬한 "실시간" 피드백. 사용자가 코드가 실제로 실행되고 있음을 체감하게 함.
- AI 스트리밍 중 설명 카드가 **위에서 아래로 채워지는** 연출이 스토리텔링 효과를 줌.
- 재생 컨트롤은 모든 청크 완료까지 비활성으로 두어 "중간에 탐색하다 설명 없는 step을 보는" 혼란을 방지.
