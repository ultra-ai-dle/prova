# Prova — Screen 05: Data Exploration Mode (데이터 탐색 모드)

## 목적
AI Phase 1 또는 Phase 2가 네트워크 에러 등으로 실패했을 때의 폴백 모드.
Pyodide 실행은 성공했으므로 rawTrace는 정상 보유하고 있다.
AI 시각화(GRID/LINEAR 등) 없이도 **변수 K-V 테이블 + 타임라인**으로
기본 디버거처럼 동작한다.

---

## 두 가지 진입 시나리오

| 시나리오 | 진입 방식 | Center 상태 |
|---|---|---|
| **Phase 1 실패** | `/api/analyze` 에러 → 토스트 + 재시도 버튼 | K-V 테이블 뷰 (strategy 미결정) |
| **Phase 2 중간 실패** | 청크 미수신 → 자동 데이터 탐색 모드 전환 | K-V 테이블 뷰 (이후 step 설명 없음) |

## 컴포넌트 상세

### 1. 에러 알림 배너 (amber 스트립)
- 화면 최상단, 44px 높이 amber 배너(`#7c4a00` 배경, `#e3b341` 텍스트).
- 텍스트: "⚠️ AI 연결에 실패했습니다. 기본 변수 뷰로 코드 흐름을 추적합니다."
- 우측에 작은 버튼: "↻ AI 분석 재시도" (아웃라인 스타일, amber 계열).
  - Phase 1 실패: 클릭 시 `/api/analyze`만 재호출 (Pyodide 재실행 없음).
  - Phase 2 실패: 클릭 시 실패한 청크부터 `/api/explain` 재호출.
- 배너는 사용자가 직접 닫기 버튼(×)으로 닫을 수 있음.

### 2. 알고리즘 배지 (실패 상태)
- 회색 pill, 테두리 `#30363d`.
- 텍스트: "알고리즘 감지 실패" (흰색 텍스트 없고 회색).

---

### 3. Center Pane — K-V 테이블 뷰 (핵심 폴백 UI)

AI 시각화(GRID/LINEAR) 대신 **기본 변수 테이블**로 교체.
`rawTrace[currentStep].vars` 전체를 K-V 테이블로 렌더링.

```
┌─ 📋 변수 상태 — Step 12 / 47 ─────────────────────────────┐
│                                                             │
│  변수명          타입      값                               │
│  ─────────────────────────────────────────────────────    │
│  queue           list      [(1,0), (0,1), (2,1)]          │
│                            ↳ [(1,0), (0,1), (2,1)]  [펼치기]│
│  visited         list2d    [[T,F,...],[F,T,...],...]  [펼치기]│
│  dist            int       3                               │
│  i               int       1                               │
│  j               int       0                               │
│                                                             │
│  ─ 외부 스코프 (depth: 0) ───────────────────────────────  │
│  grid            list2d    [[0,1,...],[1,0,...]]  [펼치기]  │
│                                                             │
└─────────────────────────────────────────────────────────── ┘
```

- **헤더**: "📋 변수 상태 — Step N / 전체 M"
- **컬럼**: `변수명` | `타입` | `값` (3열)
  - 타입: `int`, `str`, `list`, `list2d`, `bool` 등 회색 작은 pill.
  - 값: 스칼라는 그대로 표시. 리스트/2D배열은 첫 줄 요약 + **[펼치기]** 버튼.
- **[펼치기]** 클릭 시 중첩 값을 들여쓰기로 확장 (accordion).
  - 예: `list2d`는 각 행을 줄로 표시.
- **변경된 변수**: 이전 step 대비 변경 시 노란색 배경 flash (변수 모니터와 동일한 효과).
- **외부 스코프**: `parent_frames` 기반. 구분선 + 흐릿한 배경(`opacity 60%`).
- 행 호버 시 해당 변수 배경 살짝 밝아짐.

### 4. Right Pane

#### 변수 모니터
- K-V 테이블 뷰와 연동. 이미 Center에서 보여주므로 **간략 요약** 형태 유지.
- `key_vars`가 없으므로 강조 없이 평등하게 나열.
- 타입 인식 표시: `list` → 인덱스 뷰, `dict` → 키-값 뷰.

#### AI 설명 카드 (실패 상태)
```
┌─ 현재 단계 ──────────────────┐
│  L.15                         │
│  ─────────────────────────── │
│  (설명 없음)                  │  ← 회색 이탤릭 텍스트
│  AI 연결에 실패했습니다.      │
└──────────────────────────────┘
```
- 스켈레톤 shimmer 없이 정적 회색 텍스트.
- Phase 2 부분 실패 시: 설명이 있는 step은 정상 표시, 없는 step만 "(설명 없음)".

---

### 5. Timeline Bar (정상 작동)
- rawTrace 기반으로 정상 작동.
- Dot, 현재 마커, 재생 컨트롤 모두 활성화.
- 에러 step이 있으면 빨간 ✕ Dot 표시.
- AI 설명 없이도 step 이동, Auto-play, 속도 조절 모두 가능.

## 토스트 알림

### Phase 1 실패 시
```
┌─────────────────────────────────────────────────┐
│  ⚠️ AI 분석에 실패했습니다.                      │
│     잠시 후 다시 시도해 주세요.  [재시도]         │
└─────────────────────────────────────────────────┘
```
- 우측 하단 토스트. 5초 후 자동 사라짐 (재시도 버튼 클릭 시 유지).
- 색상: amber 계열.

### Phase 2 중간 실패 시
```
┌─────────────────────────────────────────────────┐
│  ⚠️ AI 설명 연결에 실패했습니다.                │
│     변수 데이터만으로 탐색합니다.               │
└─────────────────────────────────────────────────┘
```
- 5초 자동 사라짐. 재시도 버튼 없음(자동 폴백 완료).

## UX 포인트
- AI 없이도 **기본 디버거 기능은 100% 동작**한다는 것을 명확히 전달.
- amber 배너로 "실패"가 아닌 "제한 모드"임을 인식시킴 → 사용자 이탈 방지.
- K-V 테이블의 **[펼치기]** 인터랙션으로 중첩 자료구조도 탐색 가능.
- "AI 분석 재시도" 버튼이 배너에 상시 노출되어 AI 복구를 언제든 시도 가능.
- Right Pane AI 설명 카드의 "설명 없음" 회색 텍스트는 조용하게 상태를 알림 — 빨간 에러 표시 없이.
