# Prova — PPT 마스터 프롬프트
> **용도:** Gamma, Canva, Beautiful.ai, ChatGPT 슬라이드 등 PPT AI 사이트에 이 파일 전체를 붙여 넣고 생성 요청.
> **슬라이드 수:** 최대 12장. 장당 불릿 최대 4개. 코드 블록 없이 한 줄 요약만.

---

## ▶ AI에게 전달할 프롬프트 (여기서부터 복사)

---

아래 내용을 바탕으로 **팀 내부 기술 기획 발표용 프레젠테이션(12장 이하)**을 만들어 주세요.

**발표 상황:** 바이브코딩 해커톤 팀 내부 공유. 기존 합의 기획(row 버전)에서 무엇이 바뀌었고 왜 그 결정을 했는지를 설명하는 자리입니다.
**청중:** 개발자 + 기획자 혼합. 기술 깊이는 "왜"에 집중, 코드 수준은 생략.
**톤:** 간결하고 자신감 있게. 결정의 근거를 명확히.

---

### 슬라이드 구성 지시

**슬라이드 1 — 표지**
- 제목: Prova: AI 기반 알고리즘 시각화 디버거
- 부제: 기획 설계 확정안 — 무엇을 바꿨고 왜 바꿨는가
- 슬로건: "코드를 실행하면, AI가 그 흐름을 이야기로 만듭니다."

---

**슬라이드 2 — 서비스 한 줄 정의**
- Prova가 해결하는 문제: 알고리즘 코드가 왜 틀렸는지, 어디서 에러가 났는지 시각적으로 이해하기 어렵다
- 핵심 가치: 브라우저에서 Python 코드를 직접 실행 → AI가 실행 흐름을 단계별 시각화 + 자연어 해설로 변환
- 타겟: 코딩 테스트 준비생 / 복잡한 로직을 빠르게 검증하려는 개발자

---

**슬라이드 3 — 초안(row) vs 확정안(prova): 핵심 전환**

이 슬라이드는 2열 비교표로 구성하세요.

| 항목 | 초안 (row) | 확정안 (prova) |
|------|-----------|---------------|
| 실행 방식 | LLM이 코드를 논리 시뮬레이션 (Zero-Engine) | Pyodide(WebAssembly)가 실제 Python 실행 |
| Trace 생성 | AI가 100% 생성 → 환각 위험 | sys.settrace 훅으로 100% 정확 수집 |
| AI 역할 | 실행 + 설명 모두 담당 | 설명 + 시각화 지시만 담당 (역할 분리) |
| 슬로건 | "실행하지 말고 시각화로 읽으세요" | "코드를 실행하면 AI가 흐름을 이야기로" |

핵심 메시지: **"AI는 해석에 집중하고, 실행은 엔진이 한다"**

---

**슬라이드 4 — 왜 Zero-Engine을 버렸는가**

결정의 근거를 3불릿으로 설명:
- AI 환각: LLM은 복잡한 반복문·재귀에서 변수를 잘못 계산하는 치명적 한계
- 검증 불가: AI가 생성한 Trace가 실제 실행과 일치하는지 확인할 방법 없음
- 해결: Pyodide(WebAssembly) → 브라우저에서 Python을 직접 실행, AI는 설명·시각화 지시로 역할 한정

---

**슬라이드 5 — 확정 아키텍처: 3단계 파이프라인**

세 박스(화살표로 연결)로 표현:
1. **TRACE** — Pyodide + sys.settrace (Web Worker): 코드 실행, 매 라인 변수 상태 수집
2. **AI PHASE** — Gemini Flash (Next.js API): Phase1=전략 결정, Phase2=단계별 설명 생성
3. **RENDERING** — React: mergedTrace 기반 애니메이션 시각화

핵심: 각 단계가 역할을 분리하므로 실행 정확성 + 설명 품질 + 디자인 일관성이 독립적으로 보장됨

---

**슬라이드 6 — AI 2-Phase 설계: 토큰 문제 해소 방법**

- Phase 1 `/api/analyze`: 코드 + 변수 타입 구조 → 알고리즘 감지, 시각화 전략(GRID/LINEAR/GRAPH) 결정
- Phase 2 `/api/explain`: Snapshot+Delta 청크(20 step씩) → 단계별 설명 + 시각화 지시를 스트리밍 반환
- 핵심 최적화: 전체 변수 값 대신 **타입·형태만(varTypes)** + **변경분(Delta)만** 전송 → 무료 모델(Gemini Flash)로 안정 처리
- 보안: API 키는 Next.js 서버리스 함수에서만 관리, 클라이언트 노출 없음

---

**슬라이드 7 — 새로 추가된 핵심 설계 포인트 3가지**

각 항목을 아이콘 또는 넘버링으로:
1. **var_mapping**: AI가 변수 역할(예: MAIN_QUEUE)을 사용자 코드의 실제 변수명(예: q)에 바인딩 → 시각화 오매핑 방지
2. **함수 스코프 + Breadcrumb**: 재귀 호출 시 콜스택 추적, UI에서 현재 깊이·함수명 표시 + 이전 프레임 흐릿하게 표시
3. **데이터 탐색 모드 폴백**: AI 연결 실패해도 변수 K-V 테이블로 기본 디버거처럼 동작 (서비스 중단 없음)

---

**슬라이드 8 — UI 레이아웃**

3열 레이아웃 다이어그램(텍스트 박스 3개):
- Left: Monaco 에디터 (현재 실행 줄 하이라이트)
- Center: GRID / LINEAR / GRAPH 시각화 패널 (전략 자동 선택)
- Right: 변수 모니터 + AI 단계 설명 카드
- Bottom: 타임라인 (루프·분기 Dot + 에러 앵커 X)
- 특징: AI 실패 시 Center가 K-V 테이블로 자동 교체

---

**슬라이드 9 — 사용자 흐름: 에러 발견부터 원인 파악까지**

스토리 형식 플로우(왼→오 화살표):
코드 입력 → 실행(Pyodide) → 에러 step 자동 Break → 타임라인에 빨간 X → AI가 root_cause + fix_hint 제시 → Rewind로 이전 step 역추적

핵심: 사용자가 "어디서 틀렸지?"를 디버거 없이 시각적으로 파악

---

**슬라이드 10 — MVP 범위**

포함/제외 2열 표:

| MVP 포함 | 이유 |
|---------|------|
| Pyodide Web Worker 실행 | 핵심 차별점 |
| GRID + LINEAR 시각화 | BFS·스택 데모 커버 |
| AI Phase 1·2 (분석+설명) | 핵심 가치 |
| Break & Rewind + Auto-play | 교육 경험 완성 |
| 데이터 탐색 모드 폴백 | 안정성 |

| MVP 제외 | 이유 |
|---------|------|
| GRAPH 시각화 | 시간 여유 시 추가 |
| HYBRID 멀티패널 | 동일 |
| JS/Java 지원 | 향후 확장 |

**슬라이드 12 — 데모 시나리오 & 다음 단계**

데모 2가지(불릿):
1. BFS 격자 탐색 코드 → GRID 시각화로 탐색 경로 실시간 확인
2. 인덱스 에러 코드 → 에러 step Break + AI의 원인 분석 + fix_hint

다음 단계(불릿):
- UI 구현 (docs/ui/ 스펙 완성)
- Pyodide Worker 구현
- AI API 연동 (/api/analyze, /api/explain)

## 사용 가이드

### 권장 사이트
| 사이트 | 특징 | 추천 방식 |
|---|---|---|
| [Gamma](https://gamma.app) | 자유로운 레이아웃, 마크다운 입력 지원 | 프롬프트 전체 붙여넣기 → "Generate" |
| [Canva Magic Design](https://canva.com) | 이미지 풍부, 템플릿 다양 | 슬라이드별 제목+불릿만 복사해 입력 |
| [Beautiful.ai](https://beautiful.ai) | 다이어그램 자동 정렬 강점 | 슬라이드 3, 5, 10번 특히 잘 나옴 |
| ChatGPT + Code Interpreter | 정밀 제어 가능 | "PPTX로 생성해줘" 요청 |

### 팁
- 비교표 슬라이드(3번, 10번)는 그대로 2열 표로 붙이면 레이아웃 자동 적용.
- 5번 파이프라인은 "3-step 플로우 다이어그램으로 만들어줘"를 명시하면 박스-화살표로 나옴.
- 색상 hex(`#0d1117`, `#3fb950`)를 직접 언급하면 Gamma는 반영 가능.
